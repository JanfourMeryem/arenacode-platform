Treasure Challenge

Treasure Run

Read the run data from stdin, choose the best lane on each step, then run your code. The platform generates a new coin distribution every run and replays your path.

Rules

  • Read all challenge input from stdin before printing moves.
  • Output exactly one lane choice per step using this mapping:
1 = LEFT 2 = CENTER 3 = RIGHT
  • The goal is to collect as many coins as possible.
  • Each run uses fresh generated coin rows, so hardcoded solutions will fail.

Input / Output

Injected input:

  • n = number of steps
  • start_lane = initial lane code (1/2/3)
  • then n lines: left_coins center_coins right_coins
n
start_lane
left0 center0 right0
left1 center1 right1
...

Output: exactly n lines, one code per step.

2
3
3
1
2

Example

# PSEUDO-CODE (not runnable)
read n
read start lane

repeat n times:
    read coins in left, center, right lanes
    choose your lane with your own strategy
    output one value among {1, 2, 3}
